# События: тип мероприятия, теги и программа

Документ для **заказчика и контент-команды**: как устроена классификация и «содержание» события без отдельного справочника тематик — программа задаётся **сегментами**, для витрины и фильтров используются **тип** и **теги**.

---

## Коротко

| | **Тип события** | **Теги** | **Сегменты программы** |
|---|------------------|----------|-------------------------|
| **Вопрос** | Какого *формата* мероприятие? | Как его *пометить* для списков и поиска? | *Что происходит* по времени (доклады, перерывы)? |
| **Примеры** | Митап, воркшоп, конференция, нетворкинг | Meetup, тренды, город | Блок «Доклад», «Кофе-брейк» со временем и спикерами |
| **Сколько** | Одно значение на событие | Несколько | Много записей на одно событие |
| **Где важны** | Бейджи, карточки, понятный формат | Фильтры API, поиск, витрина | Страница события, расписание дня |

Отдельной сущности «тематика мероприятия» в модели нет: смысл дня раскрывается **сегментами** и текстом; **теги** закрывают тематические срезы для сайта и отборов.

---

## Тип события

Стабильная метка **вида** мероприятия (митап, воркшоп, конференция, нетворкинг). Удобна для подписей на сайте, единой логики карточек и отчётов по формату. Список типов меняется редко и согласуется с разработкой.

---

## Теги

Общий справочник из приложения **core**: одни и те же теги могут быть у событий и новостей. Используются для фильтрации в API (`tag`, `tags`), поиска и маркировки на витрине.

---

## Сегменты программы (EventSegment)

Элементы **расписания** конкретного события: название, время, место, описание, привязанные спикеры. Это и есть детальная «структура» дня; дублировать её отдельным справочником тематик не требуется.

---

*Актуальные модели: `apps/events/models.py` — `Event`, `EventSegment`, связь с `Tag` из core.*
