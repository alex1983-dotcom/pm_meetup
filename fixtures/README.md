# Один общий файл фикстур

**initial_data.json** — все тестовые данные по макетам Figma в одном файле. Бэкенд создаёт файл, фронт у себя разворачивает.

### Важно

- **`seed_data`**, не ручной `dumpdata` с локальной БД. Иначе в репозиторий попадут «чужие» пользователи, черновики и пустые связи вместо макетов Figma.
- Обновить файл в git только так: **`dump_fixtures --seed`** (см. ниже).
- **`loaddata`** вставляет записи заново. Если в БД уже были данные / сид — возможны конфликты уникальных полей. Для чистой подстановки: пустая БД или `flush` (только dev), затем `migrate`, затем `loaddata`.

---

## Бэкенд: создать фикстуры (вместе с данными)

Из корня проекта при запущенных контейнерах:

```bash
docker compose exec web python manage.py dump_fixtures --seed
```

Сначала в БД загружаются тестовые данные (`seed_data`), затем они экспортируются в **fixtures/initial_data.json**. Этот файл кладут в репозиторий или отдают фронту.

---

## Фронт: развернуть фикстуры у себя

Контейнеры запущены, миграции применены. Один файл — одна команда:

```bash
docker compose exec web python manage.py loaddata initial_data
```

После этого в БД будут теги, пользователи, события, новости, контент, страницы с блоками, материалы. Админка: admin@pm.meetup / admin123. API: заголовок `X-API-KEY: test-api-key-12345`.
